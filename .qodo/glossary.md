# Глоссарий терминов @orion76/ng-plugin

## Основные понятия

**Plugin (Плагин)** — независимый модуль, реализующий специфическую функциональность и подключаемый к основной системе без изменения её ядра.

**Plugin Definition (Определение плагина)** — конфигурационный объект, описывающий свойства, тип, идентификатор и класс плагина. Используется для регистрации и поиска.

**Plugin Manager (Менеджер плагинов)** — центральный компонент для регистрации, поиска, создания и управления экземплярами плагинов.

**Plugin Instance (Экземпляр плагина)** — объект, созданный на основе определения плагина и реализующий его логику.

**Plugin Builder (Билдер плагинов)** — компонент для создания экземпляров плагинов из определений с поддержкой DI.

**Plugin Discovery (Обнаружение плагинов)** — механизм поиска и предоставления определений плагинов менеджеру.

**Deriver (Деривер)** — вспомогательный класс для создания производных плагинов на основе базового определения.

## Angular-специфичные термины

**NgPlugin** — декоратор Angular для регистрации плагинов в системе DI.

**NgPluginBase** — базовый класс для Angular-плагинов с интеграцией в DI.

**PLUGIN_DEFINITION** — инжекшн-токен для получения определения плагина через DI.

## Интерфейсы

**IPlugin** — базовый интерфейс плагина с обязательными свойствами (type, id, label, definition).

**IPluginDefinition** — интерфейс определения плагина с метаданными и классом.

**IPluginManager** — интерфейс менеджера плагинов с методами управления.

**IPluginTemplate** — интерфейс шаблона плагина (используется в стандартном шаблоне).

## Типы плагинов

**Standard Plugin** — стандартный плагин, созданный по шаблону `templates/standard/`.

**Config-based Plugin** — плагин, создаваемый из конфигурации через шаблон `templates/from-config/`.

## Архитектурные слои

**Infrastructure Layer (Инфраструктурный слой)** — базовый слой с техническими компонентами (DI, менеджеры, билдеры).

**Enterprise Business Logic Layer (Слой бизнес-логики предприятия)** — слой с доменной логикой, реализованной через плагины.

## Сокращения

**DI** — Dependency Injection (внедрение зависимостей)

**CMF** — Content Management Framework

**ADR** — Architecture Decision Record

**API** — Application Programming Interface

**TS** — TypeScript

**NG** — Angular

**npm** — Node Package Manager

**tb** — сокращение для `test:browser`

**bw** — сокращение для `build:watch`

**ip** — сокращение для `install:plugin`
